# 코드 컨벤션

## 커밋 메시지

conventional commits 형식, 한국어:
- `feat: 새 기능`
- `fix: 버그 수정`
- `refactor: 리팩토링`
- `test: 테스트 추가/수정`
- `docs: 문서`
- `style: 포맷팅 (코드 변경 없음)`
- `chore: 빌드/설정`

## 브랜치 이름

- `feat/{이슈번호}-{짧은-설명}`
- `fix/{이슈번호}-{짧은-설명}`

## PR 규칙

- 제목: 한국어, conventional commits, 50자 이내
- 본문: 변경 요약 + `Closes #이슈번호` + 테스트 방법

## 코드 품질

- 요청된 변경만 구현 (over-engineering 금지)
- 변경하지 않은 코드에 주석/타입 추가 금지
- 비슷한 코드 3줄이 성급한 추상화보다 나음
- 사용하지 않는 코드는 완전히 삭제

## 보안

- API 키, 비밀값 하드코딩 절대 금지
- .env 파일 커밋 금지
- 시스템 경계에서만 입력 검증
