# 이슈 품질 채점 기준

10점 만점. pipeline.yml의 `scoring.issue.threshold` (기본 8점) 이상이면 통과.

## 버그 (type/bug)

| 항목 | 점수 |
|------|------|
| 기본 (제목+설명 있음) | 2점 |
| 재현 방법 (steps to reproduce) | +3점 |
| 기대 동작 vs 실제 동작 | +2점 |
| 에러 메시지/로그 | +1점 |
| 환경 정보 (OS, 브라우저, 버전) | +1점 |
| 발생 빈도 (항상/가끔/한번) | +1점 |

## 기능 요청 (type/feature)

| 항목 | 점수 |
|------|------|
| 기본 (제목+설명 있음) | 2점 |
| 동기/이유 (왜 필요한가) | +3점 |
| 수락 기준 (언제 완료인가) | +3점 |
| 범위 정의 (무엇이 포함/제외) | +1점 |
| 목업/디자인 참고 | +1점 |

## 개선 (type/enhancement)

| 항목 | 점수 |
|------|------|
| 기본 (제목+설명 있음) | 2점 |
| 현재 동작 설명 | +3점 |
| 원하는 동작 설명 | +3점 |
| 변경 이유 | +1점 |
| 하위호환성 고려 | +1점 |

## 문서 (type/docs)

| 항목 | 점수 |
|------|------|
| 기본 (제목+설명 있음) | 4점 |
| 대상 페이지/섹션 특정 | +3점 |
| 현재 문제점 설명 | +2점 |
| 수정 방향 제안 | +1점 |

## 질문 (type/question)

| 항목 | 점수 |
|------|------|
| 기본 (제목+설명 있음) | 5점 |
| 질문이 명확하고 구체적 | +2점 |
| 이미 시도한 것 기술 | +2점 |
| 관련 코드/문서 참조 | +1점 |
