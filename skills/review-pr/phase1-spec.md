# Phase 1: 스펙 준수 검증

## 검증 항목

### 1. 구현 완성도
- 구현 계획의 모든 단계가 수행됐는가?
- 계획된 파일이 모두 변경됐는가?
- 누락된 구현이 있는가?

### 2. 범위 준수
- 구현 계획에 없는 불필요한 변경이 포함됐는가?
- 불필요한 리팩토링이 끼어들었는가?
- 변경하지 않은 파일에 스타일/주석 수정이 있는가?

### 3. 테스트 이행
- 테스트 계획에 명시된 테스트가 작성됐는가?
- 테스트가 모두 통과하는가?
- 기존 테스트가 깨지지 않았는가?

### 4. PR 형식
- 커밋 메시지가 conventional commits 형식인가?
- PR 본문에 `Closes #이슈번호`가 있는가?
- PR 제목이 한국어, 50자 이내인가?

## Phase 1 통과 기준

모든 항목이 충족되면 Phase 2로 진행.
하나라도 실패하면 해당 항목을 리뷰 결과에 명시.
