# 리뷰 체크리스트 + 점수 산출

## 체크리스트 생성

pipeline.yml의 `checklists.review`를 읽어서 체크박스 형태로 출력:

```markdown
- [x] 코드 컨벤션 준수 ✅ (20/20)
- [x] 보안 취약점 없음 ✅ (25/25)
- [ ] 테스트 커버리지 충분 ⚠️ (15/25)
- [x] 성능 이슈 없음 ✅ (15/15)
- [x] 문서 변경사항 반영 ✅ (15/15)
```

## 점수 산출 공식

1. 각 항목 만점에서 감점 방식
2. 감점 사유를 구체적으로 기록
3. 항목별 점수 합산 → 총점 (100점 만점)
4. pipeline.yml의 `scoring.review.verdicts`에 따라 판정

## 판정 출력 형식

### 머지 가능 (90~100)
```
✅ **머지 가능** — 코드 품질이 우수합니다.
```

### 승인 권장 (70~89)
```
👍 **승인 권장** — 사소한 개선점이 있지만 머지해도 됩니다.
개선 사항: {목록}
```

### 수정 필요 (50~69)
```
⚠️ **수정 필요** — 다음 사항을 수정해주세요:
🔴 {critical 항목들}
🟡 {warning 항목들}
```

### 주요 문제 (0~49)
```
❌ **주요 문제** — 다음 문제를 해결해야 합니다:
🔴 {critical 항목들}
```
